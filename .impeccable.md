# Design Context — ImmoRadar

## Users

**Professional real estate investors** scanning the Austrian market daily for deals. They open this app as part of their morning workflow — checking new listings, score changes, alert matches. They expect institutional-grade tools with zero friction. Time is money; every interaction should surface insight, not require hunting.

**Context:** Desktop-first (macOS 14+), likely on a large display, possibly alongside a browser and spreadsheet. Sessions range from quick 2-minute alert checks to deep 30-minute analysis dives.

## Brand Personality

**Precise · Calm · Confident**

The app speaks with quiet authority. It doesn't shout, animate excessively, or use color for decoration. Every visual element earns its place by communicating data or status. The tone is that of a trusted analyst who presents findings clearly and lets the numbers speak.

## Emotional Goal

**"I have an edge."** — Quiet confidence. The user should feel that this tool gives them an information advantage. Everything is under control, nothing is missed, the data is reliable.

## Aesthetic Direction

- **Native macOS** — San Francisco type, system colors, standard controls. Never feel like a web app in disguise.
- **Apple Finance/Stocks influence** — Clean data presentation, subtle sparklines, restrained color, typographic hierarchy does the heavy lifting.
- **Fantastical/Things 3 influence** — Polished indie Mac craft. Warm professionalism. Attention to detail in spacing, transitions, and micro-interactions that feel considered, not flashy.
- **System-adaptive** — Full light/dark mode support via macOS system colors. Both modes are first-class citizens.

## Anti-References

1. **Cluttered dashboards** — No competing charts, no widget overload. Every metric earns its space.
2. **Generic web apps** — No Electron-wrapper feel, no Material Design, no non-native controls.
3. **Consumer real estate portals** — Not Zillow/Immoscout. No photo carousels, no playful colors, no casual tone.
4. **Enterprise SaaS** — No Salesforce tabs, no breadcrumb trails, no modal-heavy flows, no corporate gray.

## Design Principles

### 1. Data Density Through Clarity, Not Compression
Show more information by arranging it intelligently — 2-column grids, inline indicators, semantic color — never by shrinking text or cramming elements. Full headings, readable fonts, proper spacing always.

### 2. Every Color Communicates
No decorative color. Blues, greens, oranges, and reds map to scores, status, and confidence. Backgrounds use system neutrals. If something is colored, it means something.

### 3. Native or Nothing
Use SwiftUI system components, San Francisco typography, macOS interaction patterns. The app should feel like it ships with the OS. No custom chrome that fights the platform.

### 4. Typography Is the Design
Hierarchy comes from font weight, size, and spacing — not from borders, backgrounds, or color. Monospaced digits for numbers. Rounded design for summary values. System fonts throughout.

### 5. Quiet Until Relevant
Animations are functional (state transitions, loading). No gratuitous motion. Respect Reduce Motion. Alerts and scores use color to draw attention only when the data warrants it.

## Design Tokens (Source of Truth)

All visual values live in `apps/macos/ImmoRadar/DesignSystem/Theme.swift`:

- **Spacing:** 2 / 4 / 8 / 12 / 16 / 24 / 32 / 48
- **Radii:** 4 / 8 / 12 / 24
- **Shadow:** black 6% opacity, radius 2, y-offset 1
- **Score colors:** blue (80+), green (60+), orange (30+), red (0+)
- **Typography:** System San Francisco, dynamic sizing, `.monospacedDigit()` for numbers
- **Backgrounds:** `NSColor.controlBackgroundColor` (adaptive)

## Locale

Austrian German (`de_AT`). All dates as `dd.MM.yyyy`, prices as `EUR 299.000`, areas as `72,5 m²`, decimal comma throughout. Formatting centralized in `PriceFormatter.swift`.

## Accessibility

macOS system defaults: VoiceOver support, Dynamic Type, Reduce Motion respect. No additional requirements beyond platform standards.

## Component Library

23 purpose-built design system components in `apps/macos/ImmoRadar/DesignSystem/`. Key patterns:
- `CardModifier` — consistent card styling (padding lg, radius lg, subtle shadow)
- `ScoreIndicator` — circular score display (3 sizes: 32/44/80)
- `StatusBadge` — colored dot + text in capsule
- `MetricCell` — label-over-value for data grids
- `SparklineView` — 72×24 inline trend charts
- `FlowLayout` — text-wrapping tag layout
